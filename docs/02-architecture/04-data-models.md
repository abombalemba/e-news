# Модели данных

## Источник (PressCenter)

Сущность, описывающая один сайт или организацию, из которой собираются новости.

Поля:
- идентификатор (id) — уникальный UUID для хранения в базе данных
- название (title) — отображаемое имя источника
- тег (ticker) — короткое обозначение, используемое для фильтрации
- категория (field) — принадлежность к группе новостей
- URL (url-official) — адрес главной страницы

## Лента (PressCenterFeed)

Сущность, описывающая конкретную страницу или RSS-поток внутри источника. Один источник может содержать несколько лент.

Поля:
- идентификатор (uuid) — уникальный UUID
- идентификатор источника (uuid-press-center) — ссылка на родительский источник
- активность (use) — признак, включена ли лента в обработку
- URL (url) — адрес для парсинга
- последний заголовок (last-title) — для обнаружения новых публикаций
- время последней публикации (last-time) — для отслеживания новых материалов
- последняя ссылка (last-link) — URL последней новости
- временная метка проверки (timestamp) — когда система последний раз обращалась к источнику

## Пользователь (User)

Сущность, описывающая подписчика в конкретном канале доставки.

Поля:
- идентификатор пользователя в мессенджере (user-id)
- имя пользователя (username)
- имя и фамилия (first-name, last-name)
- список исключений (excluded) — теги (ticker) и категории (field), которые пользователь не хочет получать

## Файл парсера

Каждая лента связана с файлом, содержащим код для извлечения данных. Файл хранится в файловой системе и идентифицируется по UUID ленты.

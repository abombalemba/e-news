# Компоненты системы

## Ядро

Ядро является центральным элементом, отвечающим за управление процессом сбора новостей. Оно работает в бесконечном цикле, периодически инициируя проверку источников.

Функции ядра:

- получение актуальных источников
- запуск задач парсинга для каждого активного источника
- прием результатов от сервиса парсинга
- сравнение новых данных с ранее полученными
- отправка новых данных в брокер сообщений
- обновление состояния источников в базе данных

Ядро не выполняет парсинг непосредственно. Оно делегирует эту задачу специализированным компонентам и ожидает от них структурированный ответ.

## Сервис парсинга

Сервис парсинга состоит из набора .py файлов, каждый из которых предназначен для работы с конкретным источником. Модули изолированы друг от друга и могут быть написаны с использованием различных подходов в зависимости от структуры целевого сайта.

Каждый модуль:
- выполняет HTTP-запрос к источнику
- извлекает данные с использованием HTML-парсинга или RSS-ленты
- возвращает результат в унифицированном формате

Модули работают независимо. Сбой одного модуля не влияет на работу остальных.

## Брокер сообщений

Брокер обеспечивает асинхронную передачу данных между компонентами. Все взаимодействия в системе проходят через брокер, что исключает прямые зависимости между сервисами.

Топики брокера:
- каналы для рассылки новостей

Брокер хранит сообщения до момента их успешной обработки. Если получатель временно недоступен, сообщение не теряется и будет доставлено при восстановлении соединения.

## Хранилище данных

Хранилище содержит:
- конфигурацию источников
- пользовательские данные и подписки

Хранилище используется ядром для планирования работы и каналами рассылки для фильтрации новостей перед отправкой.

## Каналы доставки

Каналы доставки реализованы в виде ботов для мессенджеров. Каждый бот:

- поддерживает соединение с брокером сообщений
- получает новые новости из соответствующего топика
- проверяет настройки каждого пользователя
- отправляет новости подписчикам с учетом их фильтров
- обрабатывает команды пользователей для управления подпиской

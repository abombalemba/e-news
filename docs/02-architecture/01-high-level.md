# Высокоуровневая архитектура

## Общая структура

Система представляет собой набор независимых компонентов, взаимодействующих через брокер сообщений. Каждый компонент выполняет строго определенную функцию и может быть заменен или масштабирован независимо от остальных.

## Компоненты

Ядро управляет процессом сбора новостей. Оно инициирует проверку источников по расписанию, обрабатывает результаты и отправляет новые данные в брокер.

Сервис парсинга состоит из набора файлов, каждый из которых предназначен для работы с конкретным источником. Модули изолированы и выполняют только одну функцию — извлечение данных из заданного источника.

Брокер сообщений обеспечивает асинхронную передачу данных между компонентами. Все взаимодействия в системе проходят через брокер, что исключает прямые зависимости между сервисами.

Хранилище данных содержит конфигурации источников и настройки пользователей. Используется ядром для планирования работы и каналами рассылки для фильтрации.

Каналы доставки реализованы в виде ботов для мессенджеров. Они получают новости из брокера, фильтруют их согласно настройкам пользователей и отправляют подписчикам.

## Взаимодействие

Ядро читает список активных источников из хранилища. Для каждого источника исполняется свой файл для парсинга. Модуль возвращает результат. Ядро сравнивает с сохраненным состоянием. При обнаружении новых данных ядро обновляет хранилище и отправляет новость в брокер. Каналы расслылки получают новость, проверяют настройки пользователей и отправляют тем, кому новость интересна.

## Ключевые принципы

Асинхронность. Компоненты не ждут ответа друг друга. Сообщения передаются через брокер и обрабатываются по мере возможности.

Слабая связанность. Компоненты знают только о брокере и хранилище, но не о других компонентах напрямую.

Отказоустойчивость. Сбой одного компонента не останавливает работу системы. Сообщения в брокере сохраняются до успешной обработки.

Масштабируемость. Каждый компонент может быть запущен в нескольких экземплярах.

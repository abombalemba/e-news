# Потоки данных

## Поток сбора

Ядро читает список активных лент из хранилища. Для каждой ленты определяется соответствующий модуль парсинга. Модуль загружается и выполняется. Результат возвращается ядру. Ядро сравнивает полученные данные с сохраненными. При обнаружении изменений обновляется состояние ленты, а новость отправляется в брокер.

## Поток доставки

Новость поступает в брокер. Каждый канал доставки получает уведомление. Канал загружает из хранилища список пользователей и их настройки. Канал последовательно проверяет каждого пользователя. При совпадении новость отправляется пользователю. При несовпадении пользователь пропускается.

## Поток управления

Пользователь отправляет команду боту. Бот определяет тип команды и параметры. Бот обращается к хранилищу для чтения или обновления настроек пользователя.

## Состояния ленты

Активная — лента включена и регулярно проверяется.

Неактивная — лента отключена вручную.

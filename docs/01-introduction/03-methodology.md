# Методология разработки

## Подход

Разработка ведется в рамках гибридной методологии, сочетающей элементы Scrum и Kanban. Такой подход позволяет совместить плановую работу над функциональностью с оперативным реагированием на проблемы, возникающие при поддержке парсеров.

## Жизненный цикл проекта

#### Фаза инициации

На этом этапе сформулирована проблема, определена целевая аудитория и выбраны технологии. Создана прототипная архитектура, определены основные сущности и их взаимосвязи.

#### Фаза минимально жизнеспособного продукта

Реализована базовая функциональность: ядро системы, несколько парсеров для тестовых источников, один канал рассылки. Система способна собирать новости и отправлять их подписчикам.

#### Фаза расширения функциональности

Добавляются новые каналы рассылки, внедряется многопоточность, реализуется система фильтрации для пользователей. Проводится оптимизация производительности.

#### Фаза стабилизации и масштабирования

Планируется решение технических ограничений (капча, производительность), внедрение мониторинга, настройка автоматического развертывания.

#### Фаза эксплуатации и развития

Постоянная поддержка существующих источников, добавление новых, сбор обратной связи и улучшение функциональности.

## Организация работы

#### Планирование

Задачи группируются в спринты n-продолжительности. В начале каждого спринта определяются цели, ставятся задачи и распределяется работа.

#### Управление задачами

Открытые задачи ведутся в системе трекинга. Каждая задача имеет описание, оценку сложности и статус. Приоритеты пересматриваются регулярно.

#### Контроль версий

Исходный код хранится в репозиториях. Основная ветка всегда содержит стабильную версию. Разработка ведется в отдельных ветках с последующим слиянием.

#### Документирование

Все значимые решения фиксируются в документации. Изменения в архитектуре или процессах сопровождаются обновлением соответствующих разделов.

## Компоненты проекта

#### Кодовая база

Исходный код всех компонентов системы.

#### Документация

Архитектурные решения, инструкции по установке и эксплуатации, описание моделей данных.

#### Репозиторий презентации

Текущий репозиторий, содержащий документацию и общее описание проекта.

#### Roadmap

План развития с указанием приоритетов и сроков.

## Принятые решения

#### Микросервисная архитектура

Позволяет развивать компоненты независимо и упрощает масштабирование отдельных узлов под нагрузкой.

#### Технологический стек

Язык программирования Go выбран для компонентов, требующих высокой производительности и параллельной обработки

Язык программирования Python используется для парсеров благодаря богатой экосистеме библиотек для работы с веб-страницами

#### Асинхронное взаимодействие

Легковесный брокер MQTT обеспечивает надежную передачу сообщений между компонентами без жесткой привязки.

#### MongoDB как хранилище

Документо-ориентированная база данных подходит для хранения разнородных конфигураций и легко масштабируется.

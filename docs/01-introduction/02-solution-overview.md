# Обзор решения

## Назначение

E-News представляет собой распределенную систему для автоматизированного сбора, обработки и доставки новостей из заданных источников. Система объединяет разрозненные каналы информации в единый поток и предоставляет пользователям инструменты для его настройки.

## Основные компоненты

#### Ядро

Управляет процессом сбора данных. Запускает задачи парсинга в соответствии с настройками каждого источника. Принимает результаты, сравнивает с ранее полученными данными и передает новые данные на обработку.

#### Сервис парсинга

Выполняет извлечение данных из источников. Каждый источник требует индивидуального подхода — система поддерживает как универсальные методы работы с RSS-лентами, так и специализированные парсеры HTML.

#### Брокер сообщений

Обеспечивает асинхронную передачу данных между компонентами. Позволяет разделить процессы сбора, обработки и доставки, делая систему устойчивой к временным сбоям отдельных узлов.

#### Хранилище данных

Содержит конфигурацию источников, настройки пользователей. Используется для отслеживания изменений и предотвращения повторной отправки одних и тех же новостей.

#### Каналы рассылки

Отвечают за отправку новостей конечным пользователям. Поддерживаются мессенджеры с развитой системой ботов, позволяющей автоматизировать рассылку и управление подписками.

## Принципы работы

#### Асинхронность

Все взаимодействия между компонентами проходят через брокер сообщений. Это исключает прямые зависимости между сервисами и позволяет каждому из них работать в своем темпе.

#### Модульность

Компоненты слабо связаны и могут заменяться или обновляться независимо. Добавление нового источника не требует изменений в других частях системы.

#### Масштабируемость

Нагрузка распределяется между экземплярами сервисов. При увеличении количества источников или пользователей система может быть расширена горизонтально.

#### Отказоустойчивость

Сбой отдельного компонента не останавливает работу всей системы. Брокер сообщений сохраняет данные до момента их успешной обработки.

## Процесс обработки новости

Система периодически опрашивает каждый активный источник. При обнаружении новой публикации извлекаются заголовок, время и ссылка. Данные проходят проверку на дублирование. Если новость действительно новая, она направляется в брокер для последующей доставки. Каналы рассылки получают сообщение и отправляют его подписчикам с учетом их настроек фильтрации.

## Ключевые особенности

#### Гибкая настройка источников

Каждый источник может быть включен или отключен независимо. Для него задается категория и тег, используемые при фильтрации. Периодичность опроса может настраиваться индивидуально.

#### Персонализация доставки

Пользователи могут исключать неинтересные категории или конкретные источники из своей ленты. Фильтры применяются на стороне каналов доставки перед отправкой.

#### Расширяемость

Добавление нового источника сводится к созданию файла парсера и добавлению записи в базу данных. Остальные компоненты системы не требуют изменений.

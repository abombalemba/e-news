# Ядро

## Назначение

Ядро управляет процессом сбора данных. Оно работает в бесконечном цикле, периодически инициируя проверку источников. Ядро не выполняет парсинг непосредственно, а делегирует эту задачу модулям парсинга.

## Технологии

Ядро написано на Go. Выбор обусловлен необходимостью параллельной обработки большого количества источников. Горутины позволяют запускать тысячи легковесных потоков одновременно без значительной нагрузки на систему.

## Функции

#### Управление расписанием

Ядро хранит информацию о каждом источнике и интервале его опроса. При наступлении времени проверки ядро инициирует задачу парсинга.

#### Запуск парсинга

Для каждого источника ядро запускается соответствующий файл в системе.

#### Обработка результатов

При получении результата от модуля парсинга ядро извлекает данные и сравнивает их с сохраненным состоянием источника. Сравнение выполняется по заголовку, времени публикации или ссылке в зависимости от того, какие данные доступны.

#### Обнаружение новых данных

Если сравнение показывает, что полученные данные отличаются от сохраненных, ядро фиксирует новую запись. Состояние источника обновляется в хранилище. Новые данные отправляются в брокер для дальнейшей рассылки.

## Хранение состояния

Ядро использует хранилище для сохранения информации о каждом источнике. Для каждого источника хранятся:
- статус активности
- последний полученный заголовок
- время последней публикации
- последняя ссылка
- временная метка последней проверки

## Конфигурация

Ядро настраивается через переменные окружения:
- максимальное количество параллельных задач
- параметры подключения к хранилищу
- параметры подключения к брокеру

## Запуск

Ядро запускается как отдельный сервис. При старте оно загружает список активных источников из хранилища и инициализирует их. В процессе работы ядро получает результаты парсинга и периодически инициирует новые задачи.

# Брокер сообщений

## Назначение

Брокер обеспечивает асинхронную передачу данных между компонентами системы. Все взаимодействия проходят через брокер, что исключает прямые зависимости между сервисами.

## Технологии

Используется Mosquitto — реализация протокола MQTT. Выбор обусловлен следующими причинами:
- легковесность и низкое потребление ресурсов
- поддержка качества обслуживания (QoS)
- простота настройки и эксплуатации
- надежная доставка сообщений

## Топики

В системе используются следующие топики:
- каналы рассылки - ядро публикует данные новостей

## Качество обслуживания

Для разных типов сообщений используются разные уровни QoS:

- QoS 0 — для сообщений, потеря которых некритична
- QoS 1 — для сообщений, которые должны быть доставлены хотя бы один раз
- QoS 2 — для критических сообщений, требующих точной однократной доставки

Новые данные передаются с уровнем QoS 1. Это гарантирует, что ни одна новость не будет потеряна, но допускает возможность дублирования.

## Хранение сообщений

Брокер настроен на хранение сообщений до момента их подтверждения получателем. Если получатель временно недоступен, сообщение остается в очереди и будет доставлено при восстановлении соединения.

## Подписки

Каждый компонент подписывается только на те топики, которые ему необходимы:
- каналы рассылки подписываются на свои топики для получения новостей

## Масштабирование

При росте нагрузки брокер может быть развернут в кластерной конфигурации. Это позволяет распределить нагрузку и обеспечить отказоустойчивость.

## Безопасность

Доступ к брокеру ограничен паролем. Каждый компонент использует отдельные учетные записи с правами только на необходимые топики.

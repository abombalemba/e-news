# Каналы рассылки

## Назначение

Каналы рассылки доставляют новые данные пользователям. Каждый канал соответствует конкретному мессенджеру и реализует логику отправки сообщений и управления подписками.

## Технологии

Боты написаны на Python. Используются официальные библиотеки для работы с API мессенджеров:

- telebot — для работы с Telegram
- vk-api — для работы с VK

## Общая архитектура бота

Каждый бот состоит из двух основных частей:
- обработчик сообщений брокера — получает новые данные и отправляет их пользователям
- обработчик команд — реагирует на сообщения пользователей и управляет настройками

## Получение новых данных

Бот подписывается на топик брокера, в который ядро отправляет новые данные. При появлении сообщения бот извлекает содержимое и инициирует процесс рассылки.

## Фильтрация

Перед отправкой бот проверяет настройки каждого пользователя. Фильтрация выполняется по следующим признакам:
- категория новости
- тег источника

Если пользователь исключил хотя бы один из признаков, новость ему не отправляется.

## Управление подписками

Пользователи взаимодействуют с ботом через текстовые команды. Поддерживаются следующие действия:
- просмотр текущих исключений
- добавление категории или источника в исключения
- удаление категории или источника из исключений
- получение справки по доступным командам

## Хранение настроек

Настройки пользователей хранятся в общем хранилище. Бот читает и обновляет их при обработке команд. Использование общего хранилища позволяет синхронизировать настройки между разными каналами рассылки.

## Запуск

Каждый бот запускается как отдельный сервис. При старте бот подключается к брокеру и начинает прослушивание топиков. Также запускается обработчик входящих сообщений от пользователей.

# Модули парсинга

## Назначение

Модули парсинга извлекают данные из конкретных источников. Каждый модуль соответствует одной ленте и содержит логику, специфичную для структуры конкретного сайта или RSS-ленты.

## Технологии

Модули написаны на Python. Выбор обусловлен наличием богатой экосистемы библиотек для работы с веб-страницами. Используются следующие библиотеки:

- requests — для выполнения HTTP-запросов
- bs4 — для парсинга HTML
- lxml — для обработки XML и RSS

## Структура модуля

Каждый модуль представляет собой Python-файл, который возвращает результат в унифицированном формате.

Выходные данные:
- заголовок (last-title) — текст заголовка новости
- время публикации (last-time) — дата и/или время произвольном формате
- ссылка (last-link) — URL статьи

## Типы источников

HTML-страницы

Модуль загружает HTML-код страницы, находит элементы, содержащие новостные данные, и извлекает информацию. Для поиска элементов используются CSS-селекторы.

RSS-ленты

Модуль загружает XML-документ, парсит его и извлекает элементы канала. RSS имеет стандартизированную структуру, что упрощает извлечение данных.

## Добавление нового источника

Для добавления нового источника необходимо:
- сгенерировать uuid для источника и его лент
- создать файл модуля с именем, соответствующим идентификатору ленты
- реализовать функционал парсинга HTML-страницы или RSS-ленты
- добавить запись о ленте в хранилище
- указать активный статус

После этих действий ядро начнет опрашивать источник в соответствии с его настройками.

## Хранение модулей

Модули хранятся в файловой системе в отдельной директории. Каждый файл имеет имя, соответствующее идентификатору ленты. При запуске парсинга ядро загружает файл с соответствующим в названии uuid.

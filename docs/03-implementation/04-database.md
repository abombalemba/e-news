# Хранилище данных

## Назначение

Хранилище содержит конфигурацию источников, состояние лент и настройки пользователей. Используется ядром для планирования работы и каналами рассылки для фильтрации.

## Технологии

Используется MongoDB. Выбор обусловлен следующими причинами:
- документо-ориентированная модель соответствует структуре данных
- гибкая схема позволяет изменять формат без миграций
- высокая производительность при чтении и записи
- встроенные механизмы репликации и шардирования

## Коллекции

#### Источники (PressCenter)

Содержит информацию о сайтах и организациях. Каждый документ включает идентификатор, название, тег, категорию и основной URL.

#### Ленты (PressCenterFeed)

Содержит информацию о конкретных страницах и RSS-потоках. Каждый документ включает идентификатор, ссылку на родительский источник, статус активности, URL для парсинга и состояние последней проверки.

#### Пользователи (User)

Для каждой социальной сети используется своя коллекция для хранения данных пользователей. Каждый документ включает идентификатор пользователя в мессенджере, юзернейм, имя, фамилию, список исключений.

## Операции чтения

Ядро регулярно читает список активных лент для планирования проверок. Каналы рассылки читают настройки пользователей перед отправкой новых данных.

## Операции записи

Ядро обновляет состояние лент после каждой проверки. Каналы рассылки обновляют настройки пользователей при обработке команд.

## Безопасность

Доступ к хранилищу ограничен паролем. Для приложений создаются отдельные учетные записи с правами, необходимыми только для выполнения их функций.
